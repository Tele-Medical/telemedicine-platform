# Glossary of Important Terms

## ABDM

Ayushman Bharat Digital Mission, India's digital health ecosystem initiative.

## ABHA

Ayushman Bharat Health Account, a digital health identity concept within the ABDM ecosystem.

## Audit log

A record of who did what in the system and when.

## AuditEvent

A FHIR resource used to represent important system events such as access or actions.

## Authentication

Verifying who a user is.

## Authorization

Determining what a verified user is allowed to do.

## Batch

A specific production lot of a medicine.

## Bitrate

The amount of data used per second by media such as audio or video.

## Cache Storage

A browser storage mechanism used mainly for cached responses and app assets.

## Clinical safety

The discipline of ensuring software does not create unsafe healthcare outcomes.

## Codec

Technology that compresses and decompresses media like audio or video.

## Conflict resolution

The process of handling edits made to the same data in different places or at different times.

## Consent

A patient's informed agreement for data use or sharing.

## coturn

An open-source TURN/STUN server commonly used with WebRTC.

## Degradation

Stepping down quality gracefully, such as moving from video to audio, instead of failing completely.

## DPDP Act

Digital Personal Data Protection Act, 2023.

## Encounter

A FHIR concept representing an actual healthcare interaction, such as a teleconsultation.

## Eventual consistency

A system property where different copies of data may not match immediately but are designed to converge later.

## FHIR

Fast Healthcare Interoperability Resources, a healthcare data standard from HL7.

## FHIR R4

A widely used release version of FHIR.

## Graceful degradation

A design approach where the system remains useful under worse conditions by reducing capabilities in steps.

## ICE

Interactive Connectivity Establishment, the WebRTC process for finding network paths between peers.

## IndexedDB

A browser database for structured local storage, essential for offline-first apps.

## IVR

Interactive Voice Response, a phone-based audio menu system.

## JWT

JSON Web Token, a common token format used in authentication systems.

## Last-write-wins

A conflict strategy where the newest write replaces earlier ones, often too risky for clinical data.

## Local-first

A system pattern where user actions are saved locally first and synced later.

## NAT

Network Address Translation, a networking setup that often complicates peer-to-peer connectivity.

## Offline first

A design principle where the app remains functional without internet and syncs later.

## Outbox pattern

A local queue pattern where pending changes are stored until they can be sent to the server.

## P95 latency

The response time under which 95 percent of requests finish.

## PWA

Progressive Web App, a web application that can feel app-like, work offline, and be installable.

## Provenance

Information about where a data item came from, who created it, and under what context.

## QR code token

A machine-readable code that may help identify a patient or retrieve an access context. It should not bypass authorization.

## RBAC

Role-Based Access Control.

## Service Worker

A browser background script that supports offline behavior, caching, and some background capabilities.

## Signaling

The message exchange used to set up a WebRTC connection.

## STUN

A protocol that helps a device discover its public-facing network address.

## Sync queue

A list of locally stored operations waiting to be synchronized to the backend.

## Telemedicine

Healthcare delivered remotely using communication technologies.

## Telemedicine Practice Guidelines

Indian guidelines that shape how teleconsultation should be carried out.

## TURN

A relay protocol used when direct WebRTC peer-to-peer connection is not possible.

## Versioning

Tracking record revisions so the system can detect stale edits and conflicts.

## WebRTC

A web technology for real-time audio, video, and data communication.
